## 2. Long-Tail Programmatic SEO Patterns

These are template patterns to generate pages at scale across your 21,500+ procedures.

### Pattern A: Procedure + Insurance Modifier
Generate 2 pages per procedure (43,000 pages):
```
/costs/{procedure-slug}                          → default (overview)
/costs/{procedure-slug}/without-insurance        → uninsured price focus
```

### Pattern B: Procedure + Geography
Generate per state (50) and top 100 metros (1,075,000+ potential pages — prioritize top combinations):
```
/costs/{procedure-slug}/in/{state-slug}          → e.g., /costs/mri-scan/in/texas
/costs/{procedure-slug}/in/{city-slug}-{state}   → e.g., /costs/mri-scan/in/houston-tx
```

### Pattern C: Procedure + Question Wrappers
Generate FAQ-style pages for top 500 procedures:
```
/costs/{procedure-slug}#how-much    → anchor to "How much does {procedure} cost?"
/costs/{procedure-slug}#insurance   → anchor to "Does insurance cover {procedure}?"
/costs/{procedure-slug}#factors     → anchor to "What factors affect {procedure} cost?"
```

### Pattern D: Category Hub Pages
Generate ~50-100 category pages:
```
/costs/category/{category-slug}     → e.g., /costs/category/imaging
/costs/category/{category-slug}/{state-slug}  → e.g., /costs/category/imaging/california
```

### Pattern E: Condition-Based Cost Pages
Generate pages for top 200 conditions:
```
/conditions/{condition-slug}/treatment-costs  → e.g., /conditions/torn-acl/treatment-costs
```

## 3. Competitive Analysis

### Competitor URL Structures

| Competitor | URL Pattern | Strengths | Weaknesses |
|-----------|-------------|-----------|------------|
| **NewChoiceHealth** | `/[Procedure]-Cost` (old) → `/[procedure]/cost` (new) + `/places/[state]/[city]/[procedure]` | Strong geo+procedure combos, 12,000+ facilities | Redirecting URLs (301s to Azure), limited content depth |
| **MDsave** | `/procedures/[procedure-slug]/[hash-id]` + `/procedures/[procedure]/[hash]/[state]` + `/procedures/[procedure]/[hash]/[state]/[city]` | Deep state/city pages, transactional | Hash IDs in URLs hurt readability; not pure info |
| **CostHelper** | `health.costhelper.com/[procedure].html` | Strong E-E-A-T (user comments/prices paid), 2,000+ word pages, 97K monthly visits | Subdomain structure dilutes authority, dated design, `.html` extensions |
| **FairHealth Consumer** | `/medical` (tool-based, not individual pages) | Authoritative data source, body-part navigator | No crawlable procedure pages — all behind JS tool. Zero programmatic SEO. |
| **Healthcare Bluebook** | `/cc/[org-code]` (gated behind employer login) | Deep data, quality rankings | Completely gated — no public SEO value |
| **MedicalCostDB** | `/procedures/[procedure-slug]/` | Clean URLs, with/without insurance data, year-tagged | Only 35+ procedures, thin content |

### Key Competitive Insight

**FairHealth and Healthcare Bluebook have ZERO programmatic SEO.** Their data is locked behind JavaScript tools or employer logins. This is an enormous opportunity — the two most authoritative cost data sources have essentially no organic search presence for procedure-specific queries.

CostHelper dominates organic with long-form content + user-submitted prices but has an aging site with poor UX. NewChoiceHealth has good geo pages but is in the middle of a migration. MDsave has the best URL structure for geo targeting but is transaction-focused, not information-focused.

## 4. Recommended URL Structure

### Primary Hierarchy

```
/costs/                                          → Master index (all procedures A-Z)
/costs/{procedure-slug}/                         → Procedure cost page (national)
/costs/{procedure-slug}/without-insurance/       → Without insurance variant
/costs/{procedure-slug}/in/{state-slug}/         → State-level cost page
/costs/{procedure-slug}/in/{city-slug}-{state}/  → City-level cost page
/costs/{procedure-slug}/medicare/                → Medicare-specific pricing

/costs/category/{category-slug}/                 → Category hub page
/costs/category/{category-slug}/{state-slug}/    → Category + state combo

/conditions/{condition-slug}/treatment-costs/    → Condition cost breakdown

/compare/                                        → Cost comparison tool
/guides/                                         → Editorial cost guides
/guides/how-to-save-on-{procedure-slug}/         → Savings guide per procedure
```

### URL Slug Conventions

- All lowercase, hyphen-separated
- No hash IDs, no `.html` extensions
- Procedure slugs: `mri-scan`, `knee-replacement`, `ct-scan`, `colonoscopy`
- State slugs: `california`, `texas`, `new-york`
- City slugs: `houston-tx`, `los-angeles-ca`, `new-york-ny`
- Trailing slashes for consistency

### Canonical URL Strategy

- `/costs/mri-scan/` is canonical for all MRI cost queries
- `/costs/mri-scan/in/texas/` is canonical for Texas-specific
- Avoid duplicate pages — use canonical tags for near-duplicates
- `hreflang` not needed (English-only US market initially)

## 5. Title Tag Templates

### Procedure Pages (National)

```
How Much Does {Procedure Name} Cost in 2026? | {SiteName}
```
- Example: "How Much Does an MRI Cost in 2026? | MedCosts"
- Character target: 50-60 characters

### Procedure + Without Insurance

```
{Procedure Name} Cost Without Insurance ({Year}) | {SiteName}
```
- Example: "MRI Cost Without Insurance (2026) | MedCosts"

### Procedure + State

```
{Procedure Name} Cost in {State} ({Year}) | {SiteName}
```
- Example: "MRI Cost in Texas (2026) | MedCosts"

### Procedure + City

```
{Procedure Name} Cost in {City}, {State Abbrev} ({Year}) | {SiteName}
```
- Example: "MRI Cost in Houston, TX (2026) | MedCosts"

### Category Pages

```
{Category} Costs: Compare {Count}+ Procedures | {SiteName}
```
- Example: "Imaging Costs: Compare 45+ Procedures | MedCosts"

### Condition Pages

```
{Condition} Treatment Cost Breakdown ({Year}) | {SiteName}
```
- Example: "Torn ACL Treatment Cost Breakdown (2026) | MedCosts"

### Key Title Tag Rules
- Always include the year (updatable annually — signals freshness to Google)
- Lead with the procedure name (matches "how much does X cost" queries)
- Keep under 60 characters to avoid truncation
- Include the brand name last (lowest priority for SERP real estate)

## 7. Meta Description Templates

### Procedure Pages (National)

```
The average cost of {procedure} is ${low_avg}-${high_avg} without insurance and ${insured_low}-${insured_high} with insurance in 2026. Compare prices from {facility_count}+ facilities nationwide.
```
- Example: "The average cost of an MRI is $400-$5,000 without insurance and $20-$100 copay with insurance in 2026. Compare prices from 12,000+ facilities nationwide."
- Character target: 150-160 characters

### Procedure + Without Insurance

```
{Procedure} costs ${low}-${high} without insurance in 2026. Learn how to find the lowest cash price near you and save up to {savings_pct}% on your {procedure}.
```

### Procedure + State

```
The average {procedure} cost in {state} is ${state_avg}. Compare prices at {state_facility_count} facilities — with and without insurance. Updated {month} 2026.
```

### Procedure + City

```
{Procedure} costs ${low}-${high} in {city}, {state_abbrev}. Compare {city_facility_count} local providers side by side. Cash & insurance prices updated {year}.
```

### Category Pages

```
Compare costs for {count}+ {category} procedures — from {cheapest_procedure} (${cheapest_cost}) to {priciest_procedure} (${priciest_cost}). Updated 2026 pricing data.
```

### Condition Pages

```
Full cost breakdown for treating {condition}: doctor visits, imaging, surgery, and recovery. Average total cost: ${total_low}-${total_high}. See what to expect.
```

### Key Meta Description Rules
- Always include a dollar amount (triggers rich snippet attention; users scan for numbers)
- Always include the year
- Include a CTA-like phrase: "Compare prices," "See what to expect," "Find the lowest"
- 150-160 characters max
- Never duplicate across pages

  "relevantSpecialty": {
    "@type": "MedicalSpecialty",

  "offers": {
    "@type": "AggregateOffer",
    "lowPrice": "400",
    "highPrice": "5000",

## 9. AEO (Answer Engine Optimization) Strategy

### Google AI Overviews — Current Landscape

As of 2026, 100% of treatment and procedure queries trigger Google AI Overviews (up from 45% in 2023). This means your content MUST be structured for extraction.

### AEO Content Requirements

Pages with FAQPage markup are **3.2x more likely** to appear in Google AI Overviews. To optimize:

1. **Lead with the answer.** First paragraph must contain the price range. Do not bury it.
2. **Use exact-match H2s.** "How much does {procedure} cost?" as an H2, not creative variations.
3. **Include a summary table.** Google extracts tables for AI Overviews at high rates.
4. **Cite your data source.** E-E-A-T matters for YMYL. Reference CMS data, commercial claims data, or your proprietary dataset.
5. **Mark up with FAQPage + Speakable schemas.**
6. **Include a "last updated" date.** Freshness is a ranking signal for cost queries.
7. **Keep the critical answer in the first 300 words.** AI Overviews preferentially extract from early content.

### Link Relationships

#### Every Procedure Page Links To:

#### Every Condition Page Links To:
1. **All procedures used to treat that condition** (the key value-add)
